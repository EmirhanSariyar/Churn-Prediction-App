# Contributing

Thank you for your interest in contributing to Churn Prediction App.

## General Guidelines

- keep changes focused and easy to review,
- use clear and descriptive commit messages,
- update documentation when project behavior changes,
- prefer readable and maintainable code over premature optimization.

## Development Flow

1. Clone the repository.
2. Install dependencies from `requirements.txt`.
3. Make small, isolated changes.
4. Validate the affected part of the project.
5. Open a pull request with a concise summary.
